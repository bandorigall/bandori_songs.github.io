전체 작업 순서 (개요)
================================================================

1. 제목(원어) + 유튜브 링크 채우기
   - 공식 디스코그래피와 공식 유튜브 플레이리스트를 교차검증
   - 둘이 일치하는 기준으로 제목(원어)과 링크 컬럼을 먼저 확정
   - 플레이리스트 곡 추출(곡명 한글/일본어 깨짐 방지 위해 PYTHONUTF8=1 필수):
       PYTHONUTF8=1 yt-dlp --flat-playlist --print "%(id)s\t%(title)s" [재생목록URL]
   - 교차검증은 "영상 ID" 기준으로. 기존 CSV의 링크에서 ID만 뽑아
     플레이리스트 ID 집합과 대조 → 양쪽 차집합이 곡 누락/오타를 잡아줌
     (이번 PasPare 작업에서 02-G0MAcdmc 한 곡 누락을 이 대조로 발견)

2. 제목(한국어) 채우기
   - Bestdori(한도리)에서 한글 제목을 찾아 채움
   - 단, Bestdori가 도중에 서비스 종료됨 → 그 이후 추가된 곡 중
     일본어 제목인데 한글 제목이 없는 건 직접 잘 번역해서 채움
   - (영어 전용 제목은 "제목(한국어) 표기 규칙"에 따라 비움)

3. 앨범커버 채우기
   - 뱅드림 공식 사이트에서만 가져옴 (googleusercontent 등 비공식 금지)
   - 자세한 절차는 아래 "앨범커버 수집 (공식 사이트)" 섹션 참고
   - 발매일도 공식 디스코그래피 페이지의 "発売日"을 기준으로 함
     (팬위키 날짜나 유튜브 업로드일 X — 업로드일은 발매 하루 전 선공개인 경우 있음)
   - 채운 뒤 발매일 오름차순 정렬 (다른 밴드 CSV와 동일 규칙)

4. 가사 채우기
   - 아래 "가사 수집" / "가사 오류 교정" 섹션 절차대로 진행

제목(한국어) 표기 규칙
================================================================

- 원어 제목이 "영어 전용"(한글·히라가나·가타카나·한자 없음)이면
  제목(한국어) 컬럼은 비운다. (예: Yes! BanG_Dream!, Happy Happy Party!,
  Time Lapse, B.O.F, CiRCLING, Light Delight, That Is How I Roll!,
  COMIC PANIC!!! 등)
- 원어에 일본어/한자가 포함되면 한국어 번역 제목을 채운다.
  (예: STAR BEAT!～ホシノコドウ～ → STAR BEAT! ~별의 고동 소리~)
- 일괄 정리 스크립트(파이썬):
    import csv, re, io
    jp_kr = re.compile(r'[가-힣぀-ヿ㐀-鿿]')   # 한글·가나·한자
    rows = list(csv.reader(open(p, encoding='utf-8', newline='')))
    for r in rows[1:]:
        if len(r) >= 3 and r[1] and not jp_kr.search(r[1]):
            r[2] = ''   # 영어 전용 → 한국어 제목 비움
    w = csv.writer(open(p, 'w', encoding='utf-8', newline=''), lineterminator='\n')
    w.writerows(rows)

앨범커버 수집 (공식 사이트)
================================================================

원칙: 커버는 반드시 bang-dream.com(또는 en.bang-dream.com) 공식 자켓만.
      기존에 googleusercontent URL 로 돼있으면 전부 공식으로 교체.

곡 → 릴리스 매핑:
  - 같은 싱글/앨범에 실린 곡(A면+커플링)은 같은 자켓을 공유.
    → 한 릴리스당 자켓 1번만 구하고 그 릴리스의 모든 곡에 재사용.
  - 앨범 신곡(싱글 미발매)은 그 앨범 자켓.
  - 디지털 싱글은 공식 디스코그래피 단독 페이지가 없을 수 있음
    → 수록된 앨범의 공식 자켓으로 대체 (예: PasPare るんっ♪てぃてぃー！)

자켓 URL 얻는 법:
  1) 릴리스의 공식 디스코그래피 페이지 ID 찾기
     - 구글에 "[릴리스/곡명] bang-dream discographies" 검색
       → https://bang-dream.com/discographies/{ID} 형태 URL 확보
     - 인덱스(/discographies?artist=...)는 JS 렌더라 curl 로 ID 안 나옴. 검색이 빠름.
  2) 그 페이지에서 og:image = 공식 자켓 URL:
       curl -sL -A "Mozilla/5.0 ... Chrome/120.0 Safari/537.36" \
         "https://bang-dream.com/discographies/{ID}" \
       | grep -oE 'og:image" content="[^"]+"'
  3) 구버전 릴리스 일부는 영문 음악 페이지에 자켓이 모여있음:
       https://en.bang-dream.com/music/cat_music/{밴드}/   (예: pastelpalettes)
     단 최신 릴리스는 영문 사이트 반영이 늦으니 JP og:image 우선.

검증: 채운 자켓 URL 들이 살아있는지 HTTP 200 확인.
  - curl 로 이미지 연타하면 레이트리밋으로 000(연결실패) 뜰 수 있음
    → 요청 사이 sleep 1~2초, --max-time 25 주고 확인.

가사 수집 (lyrics_collector.py)
================================================================

설치:
  pip install undetected-chromedriver beautifulsoup4

실행 (★밴드명 인자 필수★ — 빼면 기본값 morfonica 로 돌아 엉뚱한 밴드 오염):
  python lyrics_collector.py {밴드}
  예: python lyrics_collector.py pastel_palettes
  - 크롬 버전 안 맞으면 2번째 인자로 메이저 버전: ... pastel_palettes 149

동작:
  - 나무위키 각 곡 페이지에서 가사 td 자동 탐지 (br 개수 + 일/한 혼재)
  - innerHTML 파싱 → br 1개=그룹 구분, br 2개+=섹션 구분(절/후렴)
  - {밴드}_lyrics_cache.txt 에 아래 형식으로 저장:
      제목: 곡명
      URL: https://namu.wiki/w/...
      가사:
      (원어)
      (발음)
      (한국어)
      <<<END>>>
  - 이미 캐시에 있는 곡은 스킵
  - {밴드}.csv 번역가사 링크 컬럼 자동 업데이트

나무위키 페이지를 크롤러가 못 찾을 때 (URL 부터 막힐 때):
  - 구글에 "{곡제목} 나무위키" 로 검색하면 보통 상단에 해당 곡
    나무위키 문서가 바로 뜬다. 그 URL 을 쓰면 됨.
    (이번 PasPare 의 天下トーイツA to Z☆/きゅ〜まい＊flower/Y.O.L.O 도 이렇게 찾음)

나무위키에 번역 없는 경우:
  - 직접 가사 붙여넣어서 lyrics/제목(원어).txt 수동 생성
  - {밴드}.csv 번역가사 링크 컬럼에 출처 URL 수동 입력
  예시 발생 곡(Morfonica):
    - 胡蝶翔る星月夜: 번역 없음 → Claude가 발음+번역 생성
    - ビューティ・フォー: 나무위키보다 디시 번역이 더 나음 → 수동 교체
    - Tempest: 나무위키 URL이 /Tempest(BanG Dream!) 형태라 크롤러 미탐지 → 수동

가사 파일 형식 (lyrics/제목(원어).txt):
  - 한 블록 = 3줄: 일본어 원문 / 발음(한글) / 한국어 번역
  - 블록 사이는 빈 줄로 구분
  - 영어 애드립·백보컬 줄은 (원문 / 번역) 2줄 또는 원문 그대로

가사 오류 교정 (잘못된 곡 가사가 들어간 경우)
================================================================

증상: lyrics/제목.txt 안에 전혀 다른 곡 가사가 들어가 있음
      (예: Poppin'Party "Game Changer" 자리에 K-pop "Spell on me" 가사)
확인: 파일 첫 줄의 작사/작곡 크레딧과 곡 내용이 해당 곡과 맞는지 대조

교정 절차:
  1. 올바른 나무위키 페이지에서 가사 확보.
     - WebFetch는 403(Cloudflare) → 사용 금지.
     - 빠른 1곡 교정은 curl + 브라우저 User-Agent 로 우회 가능:
         curl -sL -A "Mozilla/5.0 (Windows NT 10.0; Win64; x64) \
           AppleWebKit/537.36 (KHTML, like Gecko) Chrome/120.0 Safari/537.36" \
           "https://namu.wiki/w/곡명" -o page.html
     - 대량/정식 수집은 lyrics_collector.py(undetected_chromedriver) 사용.
  2. 받은 HTML에서 가사 테이블 셀을 문서 순서대로 추출.
     (가사 영역: "가사[편집]" 다음 ~ "출처" 이전. 셀은 일/발음/한 순서로 반복.)
  3. lyrics/제목(원어).txt 를 위 "가사 파일 형식"으로 덮어쓰기.
  4. {밴드}_lyrics_cache.txt 에도 같은 오류가 있으면 함께 교정.

트러블슈팅
================================================================

증상: 발매일이 HTML에 "-"로 나옴
원인: CSV 헤더가 "발매일/커버일" (공백 없음)
해결: 헤더를 "발매일 / 커버일" 로 수정

증상: lyrics_collector.py 에서 "Invalid regular expression" 에러
원인: JS 정규식에 유니코드 문자 직접 사용 → WebDriver 전송 시 깨짐
해결: JS regex 를 \uXXXX escape 로 작성 (현재 EXTRACT_JS는 수정 완료)

증상: ChromeDriver 버전 불일치 에러
원인: undetected_chromedriver 가 최신 버전 드라이버 다운로드
해결: uc.Chrome(options=options, version_main=148) 처럼 현재 크롬 버전 명시

증상: 나무위키 403 에러 (WebFetch 사용 시)
원인: 나무위키가 Cloudflare로 봇 차단
해결: lyrics_collector.py(undetected_chromedriver) 또는 curl + 브라우저
      User-Agent 사용, WebFetch 사용 금지

증상: 가사가 있는데 크롤러가 못 찾음
원인1: 나무위키 페이지에 한국어 번역이 아직 없음 → 수동 처리
원인2: 가사가 2컬럼 테이블이 아니라 단일 td + br 구조
       → EXTRACT_JS가 br 10개 이상 + 일/한 혼재 td를 탐지하도록 설계됨

증상: 가사 파일에 다른 곡 가사가 들어가 있음
원인: 동명이곡/크롤링 오매칭
해결: "가사 오류 교정" 섹션 절차대로 올바른 나무위키 페이지에서 재수집

증상: cache_to_files.py / 파이썬 스크립트가 UnicodeEncodeError(cp949) 로 죽음
      (예: 'ー' 〜·ー 등 출력 중 illegal multibyte sequence)
원인: Windows 콘솔 기본 인코딩이 cp949 라 일본어 파일명 print 시 터짐
      (파일 쓰기는 utf-8 이라 정상, 죽는 건 화면 출력 단계 — 중간에 멈춰버림)
해결: 실행 앞에 PYTHONUTF8=1 PYTHONIOENCODING=utf-8 붙여서 재실행

증상: lyrics_collector.py 가 일부 곡만 "가사 없음/실패" 로 빠짐
      (이번 PasPare: 天下トーイツA to Z☆, きゅ〜まい＊flower, Y.O.L.O！！！！！)
원인: 나무위키 문서명이 자동 후보(원제목/NFKC/접미사)로 안 잡히거나
      가사 테이블 구조가 달라 EXTRACT_JS 미탐지
해결(수동 1곡씩):
  1) 올바른 나무위키 URL 확보 — 모르면 구글에 "{곡제목} 나무위키" 검색
  2) curl + 브라우저 UA 로 페이지 받기(403 우회)
  3) bs4 로 br 많고 일/한 혼재인 td 추출 → 일/발음/한 3줄 블록 생성
     (lyrics_collector.py 의 _bs4_fallback 와 동일 로직)
  4) {밴드}_lyrics_cache.txt 에 "제목:/URL:/가사:/...<<<END>>>" 형식으로 append
  5) cache_to_files.py 재실행 → txt + CSV 링크 동기화
