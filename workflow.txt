전체 작업 순서 (개요)
================================================================

1. 제목(원어) + 유튜브 링크 채우기
   - 공식 디스코그래피와 공식 유튜브 플레이리스트를 교차검증
   - 둘이 일치하는 기준으로 제목(원어)과 링크 컬럼을 먼저 확정

2. 제목(한국어) 채우기
   - Bestdori(한도리)에서 한글 제목을 찾아 채움
   - 단, Bestdori가 도중에 서비스 종료됨 → 그 이후 추가된 곡 중
     일본어 제목인데 한글 제목이 없는 건 직접 잘 번역해서 채움
   - (영어 전용 제목은 "제목(한국어) 표기 규칙"에 따라 비움)

3. 앨범커버 채우기
   - 뱅드림 공식 사이트에서 먼저 찾음

4. 가사 채우기
   - 아래 "가사 수집" / "가사 오류 교정" 섹션 절차대로 진행

처음 셋업 (밴드 처음 추가할 때)
================================================================

1. {밴드}.csv 컬럼 규칙
   - TJ번호, 제목(원어), 제목(한국어), 발매일 / 커버일, 앨범커버,
     원곡가수, 곡유형, 링크, 번역가사 링크
   - 주의: "발매일 / 커버일" 슬래시 양쪽에 공백 있어야 convert_html.py가 인식함

2. 링크 컬럼 = 공식 유튜브 재생목록 URL만 사용
   - yt-dlp로 목록 추출:
     yt-dlp --flat-playlist --print "%(title)s|%(url)s" [재생목록URL]
   - 추출 결과로 update_links.py 의 PLAYLIST_MAP 딕셔너리 업데이트 후 실행:
     python update_links.py

================================================================
제목(한국어) 표기 규칙
================================================================

- 원어 제목이 "영어 전용"(한글·히라가나·가타카나·한자 없음)이면
  제목(한국어) 컬럼은 비운다. (예: Yes! BanG_Dream!, Happy Happy Party!,
  Time Lapse, B.O.F, CiRCLING, Light Delight, That Is How I Roll!,
  COMIC PANIC!!! 등)
- 원어에 일본어/한자가 포함되면 한국어 번역 제목을 채운다.
  (예: STAR BEAT!～ホシノコドウ～ → STAR BEAT! ~별의 고동 소리~)
- 일괄 정리 스크립트(파이썬):
    import csv, re, io
    jp_kr = re.compile(r'[가-힣぀-ヿ㐀-鿿]')   # 한글·가나·한자
    rows = list(csv.reader(open(p, encoding='utf-8', newline='')))
    for r in rows[1:]:
        if len(r) >= 3 and r[1] and not jp_kr.search(r[1]):
            r[2] = ''   # 영어 전용 → 한국어 제목 비움
    w = csv.writer(open(p, 'w', encoding='utf-8', newline=''), lineterminator='\n')
    w.writerows(rows)

가사 수집 (lyrics_collector.py)
================================================================

설치:
  pip install undetected-chromedriver beautifulsoup4

실행:
  python lyrics_collector.py

동작:
  - 나무위키 각 곡 페이지에서 가사 td 자동 탐지 (br 개수 + 일/한 혼재)
  - innerHTML 파싱 → br 1개=그룹 구분, br 2개+=섹션 구분(절/후렴)
  - {밴드}_lyrics_cache.txt 에 아래 형식으로 저장:
      제목: 곡명
      URL: https://namu.wiki/w/...
      가사:
      (원어)
      (발음)
      (한국어)
      <<<END>>>
  - 이미 캐시에 있는 곡은 스킵
  - {밴드}.csv 번역가사 링크 컬럼 자동 업데이트

나무위키에 번역 없는 경우:
  - 직접 가사 붙여넣어서 lyrics/제목(원어).txt 수동 생성
  - {밴드}.csv 번역가사 링크 컬럼에 출처 URL 수동 입력
  예시 발생 곡(Morfonica):
    - 胡蝶翔る星月夜: 번역 없음 → Claude가 발음+번역 생성
    - ビューティ・フォー: 나무위키보다 디시 번역이 더 나음 → 수동 교체
    - Tempest: 나무위키 URL이 /Tempest(BanG Dream!) 형태라 크롤러 미탐지 → 수동

가사 파일 형식 (lyrics/제목(원어).txt):
  - 한 블록 = 3줄: 일본어 원문 / 발음(한글) / 한국어 번역
  - 블록 사이는 빈 줄로 구분
  - 영어 애드립·백보컬 줄은 (원문 / 번역) 2줄 또는 원문 그대로

================================================================
가사 오류 교정 (잘못된 곡 가사가 들어간 경우)
================================================================

증상: lyrics/제목.txt 안에 전혀 다른 곡 가사가 들어가 있음
      (예: Poppin'Party "Game Changer" 자리에 K-pop "Spell on me" 가사)
확인: 파일 첫 줄의 작사/작곡 크레딧과 곡 내용이 해당 곡과 맞는지 대조

교정 절차:
  1. 올바른 나무위키 페이지에서 가사 확보.
     - WebFetch는 403(Cloudflare) → 사용 금지.
     - 빠른 1곡 교정은 curl + 브라우저 User-Agent 로 우회 가능:
         curl -sL -A "Mozilla/5.0 (Windows NT 10.0; Win64; x64) \
           AppleWebKit/537.36 (KHTML, like Gecko) Chrome/120.0 Safari/537.36" \
           "https://namu.wiki/w/곡명" -o page.html
     - 대량/정식 수집은 lyrics_collector.py(undetected_chromedriver) 사용.
  2. 받은 HTML에서 가사 테이블 셀을 문서 순서대로 추출.
     (가사 영역: "가사[편집]" 다음 ~ "출처" 이전. 셀은 일/발음/한 순서로 반복.)
  3. lyrics/제목(원어).txt 를 위 "가사 파일 형식"으로 덮어쓰기.
  4. {밴드}_lyrics_cache.txt 에도 같은 오류가 있으면 함께 교정.

새 곡 추가 시 체크리스트
================================================================

□ {밴드}.csv 에 행 추가 (발매일 / 커버일 컬럼명 공백 주의)
□ 제목(원어)가 영어 전용이면 제목(한국어) 비우기
□ 재생목록에 영상 추가됐으면 update_links.py 의 PLAYLIST_MAP 갱신 후 실행
□ lyrics_collector.py 실행 → 나무위키에서 가사 자동 수집
  - 가사 없으면 수동으로 lyrics/제목(원어).txt 생성
  - 번역가사 링크 수동 입력
□ cache_to_files.py 실행 → txt 파일 + CSV 링크 동기화
  (HTML 재생성 불필요 — index.html 이 CSV/lyrics 를 직접 읽음)

================================================================
트러블슈팅
================================================================

증상: 발매일이 HTML에 "-"로 나옴
원인: CSV 헤더가 "발매일/커버일" (공백 없음)
해결: 헤더를 "발매일 / 커버일" 로 수정

증상: lyrics_collector.py 에서 "Invalid regular expression" 에러
원인: JS 정규식에 유니코드 문자 직접 사용 → WebDriver 전송 시 깨짐
해결: JS regex 를 \uXXXX escape 로 작성 (현재 EXTRACT_JS는 수정 완료)

증상: ChromeDriver 버전 불일치 에러
원인: undetected_chromedriver 가 최신 버전 드라이버 다운로드
해결: uc.Chrome(options=options, version_main=148) 처럼 현재 크롬 버전 명시

증상: 나무위키 403 에러 (WebFetch 사용 시)
원인: 나무위키가 Cloudflare로 봇 차단
해결: lyrics_collector.py(undetected_chromedriver) 또는 curl + 브라우저
      User-Agent 사용, WebFetch 사용 금지

증상: 가사가 있는데 크롤러가 못 찾음
원인1: 나무위키 페이지에 한국어 번역이 아직 없음 → 수동 처리
원인2: 가사가 2컬럼 테이블이 아니라 단일 td + br 구조
       → EXTRACT_JS가 br 10개 이상 + 일/한 혼재 td를 탐지하도록 설계됨

증상: 가사 파일에 다른 곡 가사가 들어가 있음
원인: 동명이곡/크롤링 오매칭
해결: "가사 오류 교정" 섹션 절차대로 올바른 나무위키 페이지에서 재수집
